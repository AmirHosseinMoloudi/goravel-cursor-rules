# Goravel Cursor Rules

A comprehensive collection of Cursor AI rules for Goravel framework development, ensuring consistent, maintainable, and high-quality Go applications.

## 🚀 Features

- **50+ Specialized Rules**: Covering every aspect of Goravel development
- **AI-Optimized**: Designed for seamless AI-LLM collaboration
- **Production-Ready**: Battle-tested patterns and best practices
- **Framework-Specific**: Tailored for Goravel framework conventions
- **Comprehensive Coverage**: From basic setup to advanced deployment

## 📁 Rule Categories

- **Core Framework**: Installation, configuration, and basic setup
- **Architecture**: Service containers, providers, and lifecycle management
- **HTTP & Routing**: Fiber driver, routing, middleware, and API development
- **Security**: Authentication, authorization, encryption, and security best practices
- **Database**: ORM, migrations, queries, and data management
- **Advanced Features**: Caching, sessions, validation, logging, and background jobs
- **Code Quality**: Human-AI friendly code, backward compatibility, and testing
- **DevOps**: Deployment, performance optimization, and infrastructure management

## 🛠️ Usage

1. Copy the `.cursor/rules/` directory to your Goravel project
2. Rules are automatically applied based on their configuration
3. Customize rules as needed for your specific requirements

## 📖 Documentation

Each rule includes comprehensive documentation with:
- Clear examples of what to do and what to avoid
- Best practices and patterns
- Performance considerations
- Security guidelines
- Testing strategies
